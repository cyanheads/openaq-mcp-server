# 0.1.2 — 2026-06-20

Framework-maintenance release: adopts `@cyanheads/mcp-ts-core` `^0.10.9`. No tool, resource, or behavior changes.

## Changed

- **`@cyanheads/mcp-ts-core` `^0.10.6` → `^0.10.9`** — picks up the [0.10.7](https://github.com/cyanheads/mcp-ts-core/releases/tag/v0.10.7)–[0.10.9](https://github.com/cyanheads/mcp-ts-core/releases/tag/v0.10.9) deltas: `ctx.content` collector for image/audio blocks, Canvas SQL gate `invalid_sql` classification with DuckDB binder detail, fresh-scaffold `devcheck` guards, and the two new static `devcheck` checks below.
- **`devcheck.config.json`** — added `packaging.pluginManifests: true`, enabling the new plugin-marketplace manifest checks against `.claude-plugin/plugin.json` and `.codex-plugin/{plugin,mcp}.json`.
- **Re-synced framework `devcheck` scripts** — `build-changelog.ts`, `check-framework-antipatterns.ts`, `check-skill-versions.ts`, `devcheck.ts`, `lint-packaging.ts`, and the new `check-dependency-specifiers.ts` (`Dependency Specifiers` gate, hard-fails on floating `latest`/`*`/pre-release dist-tags in `package.json` and `bun.lock` workspace maps).
- **Re-synced framework skills** to their `^0.10.9` versions (`add-tool`, `api-auth`, `api-canvas`, `api-config`, `api-context`, `api-errors`, `api-services`, `api-telemetry`, `field-test`, `git-wrapup`, `orchestrations`, `polish-docs-meta`, `report-issue-local`, `tool-defs-analysis`).
