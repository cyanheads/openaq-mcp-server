# 0.1.3 — 2026-06-30

## Added

- **`openaq_find_locations` lower-bound disclosure** — new `totalCountIsLowerBound` (boolean) and `notice` enrichment fields. When OpenAQ returns a `">N"` total, `totalCount` is kept as a numeric floor and flagged, so the count is never presented as exact. ([#3](https://github.com/cyanheads/openaq-mcp-server/issues/3))

## Changed

- **`openaq_list_countries` two-letter queries** — a two-character `query` is now an exact ISO 3166-1 alpha-2 code lookup (`US` → United States alone), winning over substring matches; longer queries still match `code`/`name` as substrings. ([#4](https://github.com/cyanheads/openaq-mcp-server/issues/4))
- **Tool descriptions** — removed implementation/mechanics phrasing (internal layer counts, "on our side", "internally") from the advertised descriptions on `openaq_list_countries`, `openaq_list_parameters`, `openaq_dataframe_describe`, `openaq_dataframe_query`, and `openaq_get_measurements`, plus the mirrored copy in `README.md` and `docs/design.md`. Behavior unchanged. ([#5](https://github.com/cyanheads/openaq-mcp-server/issues/5))

### Dependency bumps

- `@cyanheads/mcp-ts-core` `^0.10.9` → `^0.10.10`
- `@biomejs/biome` `2.5.0` → `2.5.1`
- `@types/node` `26.0.0` → `26.0.1`

## Fixed

- **`openaq_find_locations` distance ordering** — coordinate results are now sorted ascending by `distance` at the service layer so `results[0]` is the true nearest. OpenAQ `/v3/locations` is not distance-ordered, so a farther station could previously surface first; `bbox`/`iso` results (no center point, `distance: null`) keep upstream order. ([#2](https://github.com/cyanheads/openaq-mcp-server/issues/2))
- **`openaq_get_readings` nearest-station resolution** — auto-resolution from coordinates now pulls a candidate pool (`limit` 100) and trusts the distance-sorted `results[0]`, instead of `limit: 1` which could bind readings to a farther station. ([#2](https://github.com/cyanheads/openaq-mcp-server/issues/2))
- **`openaq_find_locations` lower-bound totals** — a `">N"` total from OpenAQ is no longer reported as an exact `totalCount`; the truncation guidance and the new `notice` field disclose that more stations match than the count shown. ([#3](https://github.com/cyanheads/openaq-mcp-server/issues/3))

## Security

- **Transitive `js-yaml` `3.14.2` → `3.15.0`** — the `audit:refresh` lock re-resolve bundled with the `^0.10.10` framework adoption clears [GHSA-h67p-54hq-rp68](https://github.com/advisories/GHSA-h67p-54hq-rp68) (moderate denial-of-service in js-yaml `<3.15.0`). `bun audit` now reports no vulnerabilities.
